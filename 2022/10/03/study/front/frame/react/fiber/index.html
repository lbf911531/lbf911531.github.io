React Fiber的第一次学习

发表于 2022-10-03 更新于 2022-10-04 分类于 前端

前言

本文章记录于vdom.md之后

1.问题

React V16.8版本之前，关于VirtualDOM的diff与更新采用的思路是——循环及递归。

而在线程中，如果递归的数据足够庞大，会导致主线程被长期占用（任务一旦开始就只能等结束，无法中断）；这将造成其他任务等待执行，从而产生页面卡顿等现象。【JavaScript是单线程】

举个例子：

循环递归函数执行过程中，此时用户需要点击按钮改变页面背景色，其结果是知道函数执行完成前，背景色仍然没有变化。

2.解决方案

  • 2.1 放弃循环递归的思路，改为循环，循环是可以被打断的
  • 2.2 将diff的大任务拆分成多个小任务，每个任务只比对当前新旧节点
  • 2.3 基于requestIdleCallback，利用浏览器线程空闲时间去执行任务，避免一个任务长时间占用主线程；并支持在下一个空闲时间续接执行后续任务

3.requestIdleCallback

概念1——动画：页面绘制是建立在静态基础上，通过一帧又一帧的快速切换，改变页面渲染效果，达到页面“动起来”的效果。（可以类比动画，或者canvas画布实现动画的思路）

概念2——帧率：当页面能够达到60帧/s时，肉眼可见动画是流畅的；一旦小于该值，能感觉得到页面的卡顿（这也是游戏里经常提到的掉帧一词）

概念3——函数【requestIdleCallback】

  • 因此，由1，2得：一般电脑的刷新频率在 16ms/帧

  • 结合浏览器可知： 在这一帧中，包含了用户的交互、js的执行、以及requestAnimationFrame的调用，布局计算以及页面的重绘等工作。

  • 该函数的回调的执行的前提条件是当前浏览器处于空闲状态。

    也就是说：如果上述工作执行完成后仍小于 16ms, 则表示这一帧有空闲时间，该空闲时间则将可以执行requestIdleCallback回调

  • requestIdleCallback(callback, { timeout: 1000 })

    表示 如果时间到了1s后，不论什么情况，该回调都将立即执行

4.具体思路（摘）

在 Fiber 方案中，为了实现任务的终止再继续，DOM比对算法被分成了两部分：

  1. 构建 Fiber (可中断)
  2. 提交 Commit (不可中断)

DOM 初始渲染: virtualDOM -> Fiber -> Fiber[] -> DOM

DOM 更新操作: newFiber vs oldFiber -> Fiber[] -> DOM

5.Fiber数据结构

type Fiber = {
  /************************  DOM 实例相关  *****************************/

  // 标记不同的组件类型, 值详见 WorkTag
  tag: WorkTag,
  // 组件类型 div、span、组件构造函数
  type: any,
  // 实例对象, 如类组件的实例、原生 dom 实例, 而 function 组件没有实例, 因此该属性是空
  stateNode: any,
	
  /************************  构建 Fiber 树相关  ***************************/
  
  // 指向自己的父级 Fiber 对象
  return: Fiber | null,
  // 指向自己的第一个子级 Fiber 对象
  child: Fiber | null,
  // 指向自己的下一个兄弟 iber 对象
  sibling: Fiber | null,
  // 在 Fiber 树更新的过程中，每个 Fiber 都会有一个跟其对应的 Fiber
  // 我们称他为 current <==> workInProgress
  // 在渲染完成之后他们会交换位置
  // alternate 指向当前 Fiber 在 workInProgress 树中的对应 Fiber
	alternate: Fiber | null,
		
  /************************  状态数据相关  ********************************/
  
  // 即将更新的 props
  pendingProps: any, 
  // 旧的 props
  memoizedProps: any,
  // 旧的 state
  memoizedState: any,
		
  /************************  副作用相关 ******************************/

  // 该 Fiber 对应的组件产生的状态更新会存放在这个队列里面 
  updateQueue: UpdateQueue<any> | null,
  // 用来记录当前 Fiber 要执行的 DOM 操作
  effectTag: SideEffectTag,
  // 存储第一个要执行副作用的子级 Fiber 对象
  firstEffect: Fiber | null,
  // 存储下一个要执行副作用的子级 Fiber 对象
  // 执行 DOM 渲染时要先通过 first 找到第一个, 然后通过 next 一直向后查找
  nextEffect: Fiber | null,
  // 存储 DOM 操作完后的副作用 比如调用生命周期函数或者钩子函数的调用
  lastEffect: Fiber | null,
  // 任务的过期时间
  expirationTime: ExpirationTime,
  // 当前组件及子组件处于何种渲染模式 详见 TypeOfMode
  mode: TypeOfMode,
};

6.1 fiber构建整体流程

/**
 *  1. 整个fiber的实现流程分为两部分 
 *   - 1.1 根据virtualDom生成fiber，拆分小任务，可随时打断
 *   - 1.2 等待所有的fiber都创建成功后，执行commit流程，将fiber对应节点挂载到页面上，不可打断
 *  2. fiber的实现是基于循环；一个fiber的生成是一个任务；
 *     每当一个fiber生成后，将询问浏览器是否有空闲时间执行下一个fiber的生成
 *     直到队列中再无任务为止
 *  3. fiber的构建，通过tag区分类别：原生，class组件，func组件，不同类型在创建stateNode的属性是
 *     生成的值不一，原生的是dom，另二者则是实例方法
 *     effectTag可以告知此次fiber构建后，fiber的操作类型：增，删，改
 *  4. 遍历root fiber的effects数组，这存放了所有下属fiber节点
 *  5. alternate属性挂载了fiber节点的上一次fiber值，diff算法正是比对当前fiber与fiber.alternate
 */
import createQueue from '../utils/createQueue';
import {
  HOST_ROOT, //  "host_root"; // 表示根节点
  HOST_COMPONENT, // "host_component"; // 表示原生标签
  CLASS_COMPONENT, // "class_component"; // 表示class组件
  FUNC_COMPONENT, //  "function_component"; // 表示函数组件
  getTag,
} from '../utils/nodeType';
import createStateNode from '../dom';

// 创建一个队列，存放所有的fiber生成任务
const taskQueue = createQueue();
// 声明当前要执行的子任务，可用来判断当前是否有子任务正在执行中
let nextUnitOfWork = null;
let pendingWork = null;

function performTask(deadline) {
  workLoop(deadline);
  /**
   * 如果上一个空闲时间内，执行了workLoop构建fiber后，
   * 仍然存在子任务
   * 则需要重新执行requestIdleCallback，告知浏览器在下一个空闲时间，继续构建fiber
   */
  if (!taskQueue.isEmpty() || nextUnitOfWork) {
    // 同时判断nextUnitOfWork 与 队列是否为空，
    // 是为了避免刚从队列中获取下一个nextUnitOfWork，时间片就不够了的情况
    // 这样，当isEmpty === true时，仍可确保还有遗留的任务能进入当前逻辑
    requestIdleCallback(performTask);
  }
};
function workLoop(deadline) {
  if (!nextUnitOfWork) {
    // 初始时，因为 nextUnitOfWork定义的是null,因此要从队列中获取第一个任务
    nextUnitOfWork = taskQueue.shift();
  }
  // 只要当前时间片还有空闲，就一直执行fiber的构建，直到当前时间片用完
  // while的存在，确保时间能最大利用（该时间段可能可以执行多个任务）
  while (deadline.timeRemaining() > 1 && nextUnitOfWork) {
    nextUnitOfWork = performUnitOfWork(nextUnitOfWork);
  }
  if (pendingWork) commitAllWork()
};

/**
 * 创建子级的fiber节点，
 * 父级fiber已经在上一轮的 performUnitOfWork 
 * 或者 taskQueue.shift() 这一步已经生成了
 * @param {*} fiber 
 */
function performUnitOfWork(fiber) {
  if (
    fiber.tag === HOST_ROOT ||
    fiber.tag === HOST_COMPONENT
  ) {
    reconcileChildren(fiber.props.children, fiber);
  } else if (fiber.tag === CLASS_COMPONENT) {
    // class组件
    reconcileChildren(fiber.stateNode.render(), fiber);
  } else if (fiber.tag === FUNC_COMPONENT) {
    // 函数组件
    reconcileChildren(fiber.stateNode(fiber.props), fiber);
  }
  /**
   * root > div > p1 > text1
   *            > p2 > text2        
   * 
   * 遍历的顺序： root,div,p1,text1,p2,text2
   */
  // 如果当前节点有子级，则返回子级作为下一个 reqIdleCb执行的任务
  // 注意：这里是找到下一个任务等下一个req执行，而不是在这里return后就继续执行了任务
  if (fiber.child) {
    return fiber.child;
  }

  // 如果当前节点没有子级，则返回当前节点的兄弟节点
  // 当兄弟节点不存在时，则返回当前父节点的兄弟节点
  // 但如果当前节点的父级，没有兄弟节点，则需要再往上定位父级的父级，避免中间被打断
  let parentFiber = fiber;
  while (parentFiber.return) {
    // 将当前的fiber节点存到effects中，待遍历结束后，根节点上的effects会存所有子fiber
    parentFiber.return.effects = parentFiber.return.effects.concat(parentFiber.effects.concat([parentFiber]))
    if (parentFiber.sibling) {
      return parentFiber.sibling
    }
    parentFiber = parentFiber.return;
  }
  // 走到这里，表示最后一个fiber任务执行完了，这里的fiber中包含了所有的fiber节点
  // 也就是，在这里将执行 fiber的第二阶段————commit
  console.log(parentFiber);
  pendingWork = parentFiber;
};

// 构建子fiber节点
function reconcileChildren(children, lastFiber) {
  // 传入的children可能是对象也可能是数组，为了方便处理
  const _children = Array.isArray(children) ? children : [children];
  let i = 0;
  const len = _children.length;
  let prevFiberNode;
  while (i < len) {
    const virtualDom = _children[i];
    const fiberNode = {
      type: virtualDom.type,
      tag: getTag(virtualDom),
      props: virtualDom.props,
      effects: [],
      return: lastFiber,
      effectTag: "add",
    }
    // fiberNode.tag === HOST_COMPONENT ? createDOMElement(fiber) : createInstance(fiber)
    fiberNode.stateNode = createStateNode(fiberNode);

    if (i === 0) {
      lastFiber.child = fiberNode;
    } else prevFiberNode.sibling = fiberNode;

    prevFiberNode = fiberNode;
    i++;
  }

};

function commitAllWork() {
  pendingWork.effects.forEach(fiber => {
    if (fiber.effectTag === "add") {
      let parent = fiber.return;
      // class组件或函数组件的父级可能也是组件，无法调用appendChild,所以需要向上查找到tag
      // 不再是这二者的fiber节点
      while (
        parent.tag === CLASS_COMPONENT ||
        parent.tag === FUNC_COMPONENT
      ) {
        parent = parent.return;
      }
      // 组件的stateNode是func，无法追加，真正需要的是它的子节点
      if (fiber.stateNode && fiber.tag === HOST_COMPONENT) {
        parent.stateNode.appendChild(fiber.stateNode);
      }
    }
  })
}

/**
 * 入口
 * @param {*} virtualDom 
 * @param {*} container 根节点
 */
export default function render(virtualDom, container) {
  // 注册任务（virtualDom => fiber对象），从根节点开始
  const rootFiber = {
    type: "div",    // 节点类型 (元素, 文本, 组件)(具体的类型)
    tag: HOST_ROOT,   // 节点标记 (对具体类型的分类 hostRoot || hostComponent || classComponent || functionComponent)
    props: { children: virtualDom }, // 节点属性
    stateNode: container, // 节点 DOM 对象 | 组件实例对象
    effects: [],      // 数组, 存储需要更改的 fiber 对象
  }
  taskQueue.push(rootFiber);
  // 询问浏览器是否空闲，开始执行fiber的创建任务
  requestIdleCallback(performTask);
}

6.2.1 创建DOM类型

import updateDOMElement from './updateDOMElement'

export default function createDOMElement(fiber) {
  let domElement;
  if (fiber.type === "text") {
    domElement = document.createTextNode(fiber.props.textContent);
  } else {
    domElement = document.createElement(fiber.type);
    updateDOMElement(domElement,fiber.props);
  }
  return domElement;
};


export default function updateDOMElement(
  dom,
  props = {},
  oldProps = {}
) {
  Object.keys(props).forEach(key => {
    const oldPropValue = oldProps[key];
    const newPropValue = props[key];

    if (key === "children") return;
    if (key === "key") return;
    if (oldPropValue !== newPropValue) {
      if (key.slice(0, 2) === "on") {
        const event = key.slice(2).toLocaleLowerCase();
        if (!oldPropValue || newPropValue.toString() !== oldPropValue.toString()) {
          dom.removeEventListener(event, oldPropValue);
          // React中的事件采用的是事件委托，将所有事件挂载到了document.body上
          dom.addEventListener(event, newPropValue);
        }
      } else if (key === "className") {
        dom.setAttribute("class", newPropValue);
      } else if (key === "style") {
        let cssText = "";
        Object.keys(props.style).forEach(key => { cssText += `${key}:${props.style[key]};` });
        dom.style.cssText = cssText;
      } else {
        dom.setAttribute(key, newPropValue);
      }
    }
  })
}

6.3 工具函数

function getTag(virtualDom) {
  if (!virtualDom.type) {
    return HOST_ROOT;
  } else if (virtualDom.type.prototype instanceof Component) {
    return CLASS_COMPONENT
  } else if (virtualDom.type instanceof Function) {
    return FUNC_COMPONENT;
  } else {
    return HOST_COMPONENT;
  }
};

export default function createQueue() {
  const queue = [];
  return ({
    push: queue.push,
    isEmpty: () => queue.length === 0,
    shift: queue.shift,
  })
};
