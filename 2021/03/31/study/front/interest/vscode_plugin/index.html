vscode插件的一次学习

发表于 2021-03-31 更新于 2022-10-03

安装全局命令yo

=>跳转到 vscode api官网

$ npm install yo -g

生成vscode插件项目

在一个空文件夹中执行命令如下，生成插件项目模版。插件项目语言选择javaScript。

yo code


目录结构

|____jsconfig.json
|____test      // 单元测试文件夹
| |____runTest.js
| |____suite
| | |____index.js
| | |____extension.test.js
|____CHANGELOG.md
|____.vscodeignore
|____README.md
|____yarn.lock
|____package.json
|____.vscode
| |____extensions.json
| |____launch.json
|____vsc-extension-quickstart.md
|____.eslintrc.json
|____extension.js    // 插件入口

extension.js 是插件的入口文件

package.json包含插件的配置信息（比如插件命令、快捷键、菜单均在此配置）

验证草稿搭建完成

按照上述脚手架搭建的雏形，可以通过在键入F5，然后Ctrl+Shift+P，输入Hello World，便可以在编辑器右下角看到有消息弹框了。

之后，便可以基于此，改造成自己的

1.配置插件图标

修改package.json中的icon属性


2.配置右键菜单

修改package.json中的contributes属性

{
  "activationEvents": [
		"onCommand:translate.from.to",
	],
  "contributes": {
    "commands": [
      {
        "command": "translate.from.to",
        "title": "小译"
      }
    ],
    "menus": {
      "editor/context": [
        {
          "when": "editorFocus",
          "command": "translate.from.to"
        }
      ]
    }
  }
}

这样一来，就可以在右键下拉菜单中，看到注册的命令，之后编写命令对应的事件，即可

3.注册命令，编写事件

此时，需要转到extension.js 文件中，开始编写具体逻辑；在编写中，一般会用到以下方法

  • vscode.commands.registerCommand([code],callback)
  • vscode.window.activeTextEditor[获取当前激活的编辑器对象]
  • vscode.window.showQuickPick[下拉对话框]
  • vscode.window.showInformationMessage[右下角消息弹框]
  • vscode.window.activeTextEditor.edit(builder => builder.replace(…))[文本替换]

4.本地打包安装

4.1 全局安装vsce

vsce --version

控制台能够查看vsce版本，表示安装成功

4.2 打包

vsce package

注意，如果这时候有报错，需要检查一下package.json文件中是否缺少publisher属性

如果在此之后，仍然报错， 需要检查一下readme.md文档内容是否有替换成自己的

5.发布到vscode市场

  1. 首先访问 https://login.live.com/ 登录你的Microsoft账号，如果没有则先注册
  2. 然后访问 https://aka.ms/SignupAzureDevOps，如果从未使用过Azure，则会有弹框提示，点击`continue`即可
  3. 创建个人访问令牌

  1. 令牌创建成功后，本地找个地方记录
  2. 访问 https://aka.ms/vscode-create-publisher 创建发布者账号[注意:id的值和package.json中的publisher属性的值保持一致]

  1. 执行发布命令
    $ vsce publish
    此时会要求你键入personal access token for publisher xxx：

这时候，填写第三步生成的token即可

  1. 等待一段时间，即可在市场中搜索到你的插件
  2. 发布新版本时，记得修改插件的版本号
