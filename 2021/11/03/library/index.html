从零开始封装React组件库

发表于 2021-11-03 更新于 2021-12-09 分类于 前端

零、目标

  • 基于ant design 二次封装业务组件库，并建立站点，展示组件使用文档
  • 发布npm包

一、准备环境

技术选择： dumi + father-build

$ npm init
$ npx @umijs/create-dumi-lib // 默认以当前文件夹名字为项目名

生成如下目录结构

.editorconfig
.fatherrc.ts      ===> 打包配置，默认rollup打包模式，减小打包体积（可修改为babel）
.gitignore
.prettierignore
.prettierrc
.umirc.ts         ===> dumi配置
docs/             ===> 站点首页，编写md文档
package.json      ===> 入口
README.md
src/              ===> 组件库代码，（可以同时编写md文档）
tsconfig.json     
typings.d.ts      ===> ts声明文件，可以依赖避免下载相应的@types包

二、编写组件

以笔者实践的项目为例：

  1. 引入antd

    // package.json配置
{
   extraBabelPlugins: [
    [
      'babel-plugin-import',
      {
        libraryName: 'antd',
        libraryDirectory: 'es',
        style: true,
      },
    ],
  ],
}

  2. 编写组件

    // path: src/components/basic/Title/index.tsx
import React from 'react';
import { IProps } from './interface';
import './index.less';

function BasicTitle(props: IProps) {
  const { title, className, extra } = props;
  return (
    <div className={`polard-basic-title ${className}`}>
      {title}
      <span className="polard-title-extra-op">
        {extra}
      </span>
    </div>
  )
}
export default BasicTitle;

  3. 编写文档

    ---
nav:
  title: 组件    (导航栏分组)
group:
  title: 基础    (分组)
  path: /basic   (该path最终会是 /components/basic/[当前文档所在文件夹名])
  order: 1       (排序，指定当前分组在菜单栏的序号，同时指定当前文档在当前文档的序号)
---
## 标题 BasicTitle 
## 代码演示
...
## API
...

  4. 导出组件

    // path: src/index.ts
export { default as BasicTitle } from './components/basic/Title';

三、mock数据

因是业务组件，耦合了业务需要，在二次封装组件时，部分逻辑需要调用接口，因此需要下载axios。

  1. 第一步
  • yarn add axios qs @types/qs
  1. 第二步
  • 在src目录下，创建/share/httpFetch.js 文件，基于axios二次封装请求方法，如拦截请求等
  • 然后前往.umirc.ts配置 externals及alias属性，如：
    {
  externals: {
    httpFetch: path.resolve('./src/share/httpFetch.js'),
  },
  alias: {
    share: path.resolve(__dirname, './src/share'),
  },
}
    externals能够使组件库打包后，使用项目中封装的axios，
    而alias的配置既是方便组件库使用简短路径引用，同时因其路径与项目中完全一致，也方便了组件库切换httpFetch。
    （httpFetch名字，路径完全依据实际项目而定，这里的/share/httpFetch.js 完全是项目中的同名文件所在路径）
  1. 第三步
  • 此时本地启动，一定会报错，因为配置了externals，导致依赖缺少，因此需要修改启动命令，增加全局变量（cross-env），
    if (process.env.ENV_TYPE === 'native') {
  // 本地启动的时候不需要配置 externals, 站点打包时也不需要externals(npm run docs:build)
  delete config.externals;
}
  1. 第四步
  • 基于express脚手架快速创建一个后台服务，或可安装mockjs依赖，定义接口
  • 前端项目.umirc.ts配置proxy代理

四、配置多语言

思路:

=> 提供LangProvider组件，项目入口引用该组件，将当前语言环境传入LangProvider组件中;

=> 创建context上下文，各组件内部引用该context获取当前语言环境

=> 创建language文件夹，内置en.ts,zh.ts等文件，维护多语言

=> 提供intlMessage方法，各组件内部调用该方法，传入key和params，从language读取对应的文件中的实际文本

实践:

language:

// path: src/components/locale-language/en_US.ts
export default {
  'common.please.enter': 'Please enter',
  'common.please.select': 'Please select',
  'common.all': 'All',
}
// path: src/components/locale-language/zh_CN.ts
export default {
  'common.please.enter': '请输入',
  'common.please.select': '请选择',
  'common.all': '全部',
}

Provider: 

// path: src/components/locale-lan-provider/index.tsx
import React from 'react';
import LocaleContext from './context';

/**
 * localeMap: {
 *  "zh_cn": {},
 *  "en_us": {},
 *  ...
 * }
 */
export interface Locale {
  locale: string;
  localeMap?: any;
}

export interface LocaleProviderProps {
  locale: Locale;
  children?: React.ReactNode;
}

export default class LocaleProvider extends React.Component<
  LocaleProviderProps,
  any
> {
  static defaultProps = {
    locale: {},
  };

  render() {
    const { locale, children } = this.props;
    if (locale?.locale)
      window.sessionStorage.setItem('cur_locale', locale.locale);
    return (
      <LocaleContext.Provider value={locale}>{children}</LocaleContext.Provider>
    );
  }
}

// 项目使用
// <LocaleProvider locale={{locale,localeMap}}>
//   {content}
// </LocaleProvider>

intlMessage:

export function intlMessage(
  title: string,
  config: IConfig = defaultConfig,
): string {
  // 从session中取，是为了兼容调用messages方法在没有params时可以不必传递config
  const locale = window.sessionStorage.getItem('cur_locale');

  const {
    params,
    context = {
      locale: 'zh_cn',
      localeMap: defaultLocaleMap,
    },
  } = config;
  // 合并外界传入的，
  const lastLocaleMap = {
    ...defaultLocaleMap,
    ...context.localeMap,
  };
  const lang = lastLocaleMap[locale || context.locale]?.[title];
  // console.log('locale:', context.locale, locale);
  if (!lang) return title;
  if (params) {
    return lang.replace(/\{(.+?)\}/g, ($1, $2) => params[$2]);
  }
  return lang;
}

五、dva/connect（redux）

组件如需用到connect，不建议在组件库中引入dva（项目中的dva创建的全局变量与组件库中的不共通）
因此这里采用自行模拟connect，编写高阶组件，内部使用(window as any)?.g_app?._store || {}。

六、组件编写less使用global

// .umirc.ts  配置后，global方可生效
cssLoader: {
  modules: {
    getLocalIdent: (context, localIdentName, localName) => {
      if (
        context.resourcePath.includes('node_modules') ||
        context.resourcePath.includes('ant.design.pro.less') ||
        context.resourcePath.includes('global.less')
      ) {
        return localName;
      }
      const match = context.resourcePath.match(/src(.*)/);
      if (match && match[1]) {
        const antdProPath = match[1].replace('.less', '');
        const arr = slash(antdProPath)
          .split('/')
          .map((a) => a.replace(/([A-Z])/g, '-$1'))
          .map((a) => a.toLowerCase());
        return `antd-pro${arr.join('-')}-${localName}`.replace(/--/g, '-');
      }
      return localName;
    },
  },
},

六、站点首页

如果不满意dumi提供的站点首页，可以访问Antd Design Landing，编写/下载自己喜欢的站点首页模板。

注意： 打包的时候，可能会丢失站点首页的样式，通过网上查询，这里记录笔者正在使用的一种方法

1. 删除docs/Home/index.jsx文件下引入的antMotionStyle.less
2. 如果有自定义样式文件的话，记得在antMotionStyle.less 导入所有自定义的less/css文件
3. 创建src/global.less，加入`@import '../docs/Home/less/antMotionStyle.less';`语句

七、项目打包

package.json配置:

{
   "scripts": {
    "start": "cross-env ENV_TYPE=native dumi dev",
    "docs:build": "cross-env ENV_TYPE=native dumi build",
    "build": "father-build",
    "release": "npm run build && npx standard-version && npm publish",
  },
  "sideEffects": [
    "./src/**/*.less"
  ], // 防止less，css丢失
}

standard-version方便打包时，自动修改version

其他可修改的配置：

  • 修改.fatherrc.ts文件中的cjs,esm
  • 默认rollup打包模式，生成dist文件夹，该模式下体积小，但是 es6 语法
  • 修改为babel模式时，会生成lib,es文件夹，该模式下体积大，同时需要修改package.json文件
    修改如下：
"main": "lib/index.js",
"module": "lib/index.js",
"typings": "lib/index.d.ts",

tsconfig.json配置，生成index.d.ts文件

{
  "declaration": true
}
  • 发布命令
$ npm run release
  • 站点打包
    $ npm run docs:build

后续就是通过docker构建镜像，发布等操作。至于docker具体相关的操作，还待学习
