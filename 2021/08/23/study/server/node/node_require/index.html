node中require语句的第一次学习

发表于 2021-08-23 更新于 2022-10-03 分类于 后端

NodeJS模块使用的CommonJS格式，模块通过require语句加载

在了解require之前，先记录一下commonjs相关概念：

1. CommonJS

1.1 简要

每一个文件就是一个模块，拥有自己独立的作用域，变量，方法，对其他模块不可见，从而避免全局污染。CommonJS规范有规定：模块内部，module变量（对象）代表当前模块，其exports属性是对外接口。执行require语句时，其实是加载module.exports属性。

1.2 export变量

node为每一个模块提供了一个exports变量(可以说是一个对象)，指向 module.exports。这相当于每个模块中都有一句这样的命令:

var exports = module.exports

2.1 Module.prototype.require

// /internal/modules/cjs/loader.js

// Loads a module at the given file path. Returns that module's
// `exports` property.
Module.prototype.require = function(id) {
  // typeof value !== 'string',这里只是判断 id的值是否是字符串，否则抛出错误。
  validateString(id, 'id'); 
  if (id === '') {
    throw new ERR_INVALID_ARG_VALUE('id', id,'must be a non-empty string');
  }
  // 关键代码： 定义在原型链上的require方法其实质是调用了Module._load方法
  return Module._load(id, this, /* isMain */ false);
};

2.2 Module._load:

// internal/modules/cjs/loader.js

// Check the cache for the requested file.   检查请求文件的缓存。
// 1. If a module already exists in the cache: return its exports object.
//    如果一个模块已经存在于缓存中:返回它的exports对象。
// 2. If the module is native: call `NativeModule.require()` with the
//    filename and return the result.
//    如果模块是原生模块:根据文件名调用' nativmodule .require()`，并返回结果。
// 3. Otherwise, create a new module for the file and save it to the cache.
//    Then have it load  the file contents before returning its exports
//    object.
//    否则，为文件创建一个新模块并将其保存到缓存中。然后让它在返回它的导出之前加载文件内容对象。
Module._load = function(request, parent, isMain) {
  // ...
  // 这里的request就是上方函数的id，也就是 文件路径，“./index”; 
  // 从方法名可以推知：此处根据路径，解析require需要的文件名的方法
 var filename = Module._resolveFilename(request, parent, isMain);
 // .... 具体代码片段： 2.4 ,2.5 ,2.6
 if (isMain) {
    process.mainModule = module;
    module.id = '.';
  }
  // 缓存
  Module._cache[filename] = module;

  tryModuleLoad(module, filename);

  return module.exports;
};

    2.1. 判断当前路径是否是 文件，如果是，则先调用toRealPath转化真实的路径

    2.2. 如果没有读到路径上的文件，则通过tryExtensions尝试在该路径后依次加上.js，.json和.node后缀，判断是否存在，若存在则返回加上后缀后的路径

    2.3. 如果是一个目录，则调用tryPackage方法读取该目录下的package.json文件，把里面的main属性设置为filename；如果缺少扩展名，则进入 tryExtensions，尝试拼接后缀，判断是否存在

    2.4. 当main指定的文件有误时，或不存在package.json文件，则将index作为默认文件名，依次拼接后缀

    2.5. 如果没找到则进入下一次的循环，继续查找

    2.6. 若解析成功，则把解析得到的文件名cache起来，下次require就不用再次解析了，否则若解析失败，则返回false ,对应了上方 步骤1

    简而言之：

    1. 判断当前路径是否是文件
    2. 是，则确定后缀，若无后缀则尝试依次拓展.js,.json,.node
    3. 否，则查找package.json的main属性；若main的值有误，则尝试将index当作文件名

    由此，重新回到 load 方法中，继续执行:，查找缓存，判断是否内部模块，未被缓存的模块执行new Module
