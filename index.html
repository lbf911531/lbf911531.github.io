React Fiber的第一次学习

发表于 2022-10-03 更新于 2022-10-04 分类于 前端

前言

本文章记录于vdom.md之后

1.问题

React V16.8版本之前，关于VirtualDOM的diff与更新采用的思路是——循环及递归。

而在线程中，如果递归的数据足够庞大，会导致主线程被长期占用（任务一旦开始就只能等结束，无法中断）；这将造成其他任务等待执行，从而产生页面卡顿等现象。【JavaScript是单线程】

举个例子：

循环递归函数执行过程中，此时用户需要点击按钮改变页面背景色，其结果是知道函数执行完成前，背景色仍然没有变化。

2.解决方案

  • 2.1 放弃循环递归的思路，改为循环，循环是可以被打断的
  • 2.2 将diff的大任务拆分成多个小任务，每个任务只比对当前新旧节点
  • 2.3 基于requestIdleCallback，利用浏览器线程空闲时间去执行任务，避免一个任务长时间占用主线程；并支持在下一个空闲时间续接执行后续任务

3.requestIdleCallback

概念1——动画：页面绘制是建立在静态基础上，通过一帧又一帧的快速切换，改变页面渲染效果，达到页面“动起来”的效果。（可以类比动画，或者canvas画布实现动画的思路）

概念2——帧率：当页面能够达到60帧/s时，肉眼可见动画是流畅的；一旦小于该值，能感觉得到页面的卡顿（这也是游戏里经常提到的掉帧一词）

概念3——函数【requestIdleCallback】

  • 因此，由1，2得：一般电脑的刷新频率在 16ms/帧

  • 结合浏览器可知： 在这一帧中，包含了用户的交互、js的执行、以及requestAnimationFrame的调用，布局计算以及页面的重绘等工作。

  • 该函数的回调的执行的前提条件是当前浏览器处于空闲状态。

    也就是说：如果上述工作执行完成后仍小于 16ms, 则表示这一帧有空闲时间，该空闲时间则将可以执行requestIdleCallback回调

  • requestIdleCallback(callback, { timeout: 1000 })

    表示 如果时间到了1s后，不论什么情况，该回调都将立即执行

虚拟DOM是什么

发表于 2022-10-03 分类于 前端

Virtual DOM

前言

在记录Virtual Dom相关知识点之前，先附上一个问题：为什么不建议频繁操作DOM？

  • 重绘重排，这会导致CPU和内存消耗过重
  • 操作DOM，是存在“跨界”的，而“跨界”需要“收费”

把 DOM 和 JavaScript 各自想象成一个岛屿，它们之间用收费桥梁连接。——《高性能 JavaScript》

此外，摘抄一段网上描述：虽然DOM是由JavaScript实现的，但是在浏览器中都是把DOM和JavaScript分开来实现的。由于是分开实现，每一次在通过js操作DOM的时候，就需要先去连接js和DOM。

因此，既然DOM也是JS实现的一种对象，且直接操作DOM，效率很低；由是引出一种优化手段————Virtual DOM（虚拟DOM节点）。将原本直接对DOM的修改，嫁接到Virtual DOM对象上，待合适机会，与原DOM树比对，得出差异后再一次性修改DOM。从而降低了修改DOM的频率。

另，记录一段关于DOM和JS关系的描述：DOM是一个与语言无关的API（一个独立于语言，用于操作XML和HTML文档的API），别的语言也可以实现操作DOM的具体api，但是它在浏览器中是用JavaScript来实现的。

JSX

JSX，是一种 JavaScript 语法的扩展。

虽然像是在HTML中，可以编写JS代码，但本质上仍然是JS。在React代码执行之前，会经过Babel编译，比如：

总结

由上述可以得知：

  • DOM和JS没有直接关联，只是目前被JS实现；二者通信需要代价（过桥费）
  • Virtual DOM是为了提高操作DOM效率，减少重绘重排频率的产物
  • Virtual DOM的操作是对像属性的增删改查，发生在内存中；不会影响页面渲染
  • JSX本质是JS对象，在React中用于方便程序员编写，最终会经Babel编译

此外，VDom的出现，Diff算法因此显得特别重要

在整个render过程中，其实分为两步骤：

  • render

  • commit

    这里说的render函数只是开启上方两个环节的一个入口；

章节3.Fiber.md 中有提到：

​ react 的render阶段会将ReactElement对象，也就是虚拟节点对象；基于每一个ReactElement分割成一个一个的小任务，以循环的方式，在线程中随时打断与续接的过程中，将VDOM 转换成Fiber节点（这些小任务，暂时叫Fiber任务）；

当顺着rootFiber直到最终点不再有VDOM需要被转换时，进入到下一个阶段——commit

​ 而在commit阶段中，就是对上一阶段的的产物：需要被执行操作的fiber节点集合，进行相关dom操作。至此，整个由render函数开始的完整流程迎来结束。

​ 简单来说：

  • render 阶段负责创建 Fiber 数据结构并为 Fiber 节点打标记，标记当前 Fiber 节点要进行的 DOM 操作。

  • commit 阶段负责根据 Fiber 节点标记 ( effectTag ) 进行相应的 DOM 操作。

微前端qiankun之基本概念：沙箱

发表于 2021-12-09 更新于 2022-10-03 分类于 前端

前言

在学习微前端解决方案——qiankun之前，需要对“沙箱”有一个基本的概念。

沙箱，普遍的定义是指：让你的程序运行在一个隔离的环境，不对外界其他程序造成影响，它是一种安全机制。
个人理解（直白一点的说法），可以隔离变量，隔离样式。

沙箱有三种模式：

  • 快照沙箱
  • 代理沙箱（单实例）
  • 代理沙箱（多实例）

快照沙箱

/**
 * 模拟思路：
 * ------>
 * 快照沙箱：
 *    每当激活应用时，都先暂存一次当前window对象（应用运行前的），
 *      然后需要从差异集合中取出prop，对当前window设置成该应用在运行时的window对象
 *    每当失活应用时，需要比对应用运行时的window对象和应用运行前的window对象，
 *      记录差异到差异集合中，同时用运行前的window对象覆盖当前（运行时）的window对象
 * 
 * 由此得出：
 *    快照沙箱只能支持单个子应用，
 *    会污染全局对象window，
 *    另：该方案适用于不支持 Proxy的浏览器（代理沙箱proxySandbox）
 */

class SnapshotSandbox {
  constructor(global) {
    this.midSandboxDiffMap = {}; // 记录上一次的全局对象与本次全局对象的差异
    this.windowSnapshot = {}; // 快照
    this.proxy = global || window; // 当前正在使用的全局对象
  }

  inactive() {
    // 当前应用卸载时：
    for (const prop in this.proxy) {
      if (this.proxy.hasOwnProperty(prop)) {
        // 比对上一次与当前window对象，记录差值，方便激活时，回到该应用对应的window对象
        if (this.windowSnapshot[prop] !== this.proxy[prop]) {
          this.midSandboxDiffMap[prop] = this.proxy[prop];
          // 用之前暂存的变量 还原window对象
          this.proxy[prop] = this.windowSnapshot[prop];
        }
      }
    }
  };

  active() {
    // 当前应用渲染时：
    this.windowSnapshot = {};
    for (const prop in this.proxy) {
      if (this.proxy.hasOwnProperty(prop)) {
        // 激活时，需要将当前的window对象记录到临时的变量中
        this.windowSnapshot[prop] = this.proxy[prop];
      }
    };
    // 将上一次该应用的相关变量，挂到window上
    Object.keys(this.midSandboxDiffMap).forEach(diffProp => {
      this.proxy[diffProp] = this.midSandboxDiffMap[diffProp];
    });
  };
};


takeAExample();
function takeAExample() {
  const snapshot = new SnapshotSandbox();
  snapshot.active();
  window.param1 = "变量1";
  window.param2 = "变量2";

  console.log("激活后==>", "params1:", window.param1, "params2", window.param2);
  snapshot.inactive();

  console.log("失活后==>", "params1:", window.param1, "params2", window.param2);
  snapshot.active();
  console.log("再次激活后==>", "params1:", window.param1, "params2", window.param2);
};

useClickAway

useClickAway：来自[ahooks](https://ahooks.js.org/)库，用于监听目标元素外的点击事件。

举个例子：指定点击某元素之外区域可触发log（类似局部蒙版效果）

/**
 * 指定：targets以外的DOM元素都能触发callback
 * @param {fn} callback 触发的事件函数
 * @param {array<dom>|dom} targets DOM节点或DOM节点数组
 * @param {string} eventName  绑定的事件类型
 */
function useClickAway(callback, targets, eventName) {
  eventName = eventName || "click"; // 默认点击事件;
  targets = Array.isArray(targets) ? targets : [targets]; // 转化成数组统一处理

  function wrappedEvent(event) {
    // 添加事件拦截逻辑，如果event包含了targets中的成员
    if (
      targets.some(tEle => getTargetElement(tEle).contains(event.target))
    ) {
      return;
    }
    callback(event);
  };
  document.addEventListener(eventName, wrappedEvent);
  // 抛出解绑事件
  return (function remove() {
    document.removeEventListener(eventName, wrappedEvent)
  });
};

/**
 * 获取真实的DOM节点, element可能是对象，方法
 * @param {any} element 
 */
function getTargetElement(element) {
  if (!element) return window;
  if (typeof element === "function") {
    return element();
  }
  if ("current" in element) { // 这里在doc.getElement时可以忽略
    return element.current;
  }
  return element;
};

Function.prototype.useCall = function (context) {
  var that = context || window;
  var _arguments = [...arguments].slice(1);

  that.fnc = this;
  var result = that.fnc(..._arguments);
  delete that.fnc;
  return result;

Function.prototype.useBind = function (context) {
  var that = context || window;

debounce

短时间内多次触发，每当触发一次，重新计时，直到期限时间结束后开始执行

function debounce(callback,delay) {
  let timer;
  return (
    function () {
      if (timer) {
        clearTimeout(timer)
      } 
      const args = arguments;
      const context = this;
      timer = setTimeout(() => {
        callback.apply(context,args);
      },delay);
    }
  )
};

throttle

多次触发，只有当前函数执行完后，才会开始执行下一次函数

function throttle(fn,delay) {
  let timer;
  return (
    function () {
      if (timer) return;
      timer = setTimeout(() => {
        fn.apply(this,arguments);
        clearTimeout(timer);
      })
    }
  )
};

关于parseInt

parseInt(string, radix)中的入参radix是2-36 之间的整数,表示被解析字符串的基数。

而如果radix 是 undefined、0 或未指定的，则会根据基数做假定，或十六进制，或八进制，或十进制。

简言之，假设radix = 2，那么string就是一个二进制的数。

parseInt(1/0, 18); // NaN
parseInt(1/0, 19); // 18 
// detail: 1/0 === 'Infinity'  0123456789 a(10) b c d e f g h i(18)
parseInt(1/0, 19) === > parseInt('i',19);
[1,2,3].map(parseInt); // [1, NaN, NaN]

另外，parseInt('2',2) 这样的语句是会返回NaN的，因为解析不了（毕竟二进制数怎么会有2呢？）；

更不必说parseInt('3',2)了。
