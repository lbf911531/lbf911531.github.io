原型与原型链

发表于 2021-11-30 更新于 2022-10-03 分类于 前端

原型

规则一：

​ 每一个引用类型，都有一个属性__proto__，该属性值是一个普通的对象

规则二：

​ 引用类型的__proto__属性指向该引用类型的构造函数的prototype属性，即

var obj = {};
obj.__proto__ === Object.prototype; // true

var fnc = function() {}
fnc.__proto__ === Function.prototype; // true

var arr = [];
arr.__proto__ === Array.prototype; // true

规则三：

当试图获取对象的某个属性时，如果这个对象本身没有这个属性，那么它会去它的隐式原型 __proto__（也就是它的构造函数的显式原型 prototype）中寻找 

var num = new Number();
num.__proto__.constructor === num.constructor; // true
// 也可以说是： num上不存在constructor属性，转而去num的构造方法Number的原型上去找
==> num.__proto__.constructor === Number.prototype.constructor;

原型链

举个例子：

class Person {
    constructor(name,age) {
        this.name = name;
        this.age = age;
    }
    growUp() {
        this.age += 1;
    }  
}

var 张三 = new Person("法外狂徒张三",16);
张三.toString(); // "[object Object]";

可以看到实例张三调用toString，但该方法其实并未在Person中定义。(现象一)

通过追溯：

  1. 实例中无法访问的属性，首先转化成去该实例的构造方法中查找，也就是：

    Person.prototype

    然而打印结果：{constructor: ƒ, growUp: ƒ}

    不存在 toString，即现象一；

  2. 但，构造方法的原型本身也是一个对象，（而对象都有一个构造方法Object），因此该对象也会存在__proto__属性，继而：

    Person.prototype.__proto__

    打印结果：

    {
  constructor: ƒ Object(),
  toString: ƒ toString(),
  __proto__: null,
  ...
}

    而改结果又和 Object.prototype 一致，

    因此综合上述说法，可以得出以下结论：

    张三.toString()

    ===> Person.prototype.__proto__.toString

    ===> Object.prototype.toString

    形成一条原型链

  3. 此外，Object.prototype.__proto__ === null,也可以说明，引用类型的原型链的终端最终是null（ 设计上是为了避免死循环而设置的 ）

结论

由上，最终再简单总结：

  • prototype 是构造方法才有的属性
  • __proto__是构造方法的实例（每个对象） 具有的属性
  • 所有的__proto__都将指向构造方法的prototype
  • 通过层层指向，最终形成原型链，而基于原型链，可以继承来自上层的方法，或者重写
  • __proto__是浏览器为了方便我们找到 当前实例 的 构造函数的原型而加的一个属性

额外补充：

张三.contructor === Person;

Person.constructor === Function;

Object.constructor === Function;

obj.constructor === Object;

Object.constructor === Function;

node中require语句的第一次学习

发表于 2021-08-23 更新于 2022-10-03 分类于 后端

NodeJS模块使用的CommonJS格式，模块通过require语句加载

在了解require之前，先记录一下commonjs相关概念：

1. CommonJS

1.1 简要

每一个文件就是一个模块，拥有自己独立的作用域，变量，方法，对其他模块不可见，从而避免全局污染。CommonJS规范有规定：模块内部，module变量（对象）代表当前模块，其exports属性是对外接口。执行require语句时，其实是加载module.exports属性。

1.2 export变量

node为每一个模块提供了一个exports变量(可以说是一个对象)，指向 module.exports。这相当于每个模块中都有一句这样的命令:

var exports = module.exports

理由

ES6 promise的模拟

发表于 2021-07-01 更新于 2022-10-03 分类于 前端

定义

Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理且更强大。它最早由社区提出并实现，ES6将其写进了语言标准，统一了用法，并原生提供了Promise对象。

特点

  1. 对象的状态不受外界影响 （3种状态）
  • pending
  • resolved
  • rejected
  1. 一旦状态改变就不会再变 （两种状态改变：成功或失败）
  • Pending -> Fulfilled
  • Pending -> Rejected

API

方法 含义
Promise.prototype.then 链式操作
Promise.prototype.catch 捕捉错误
Promise.all 入参是数组，成员为promise实例，仅当所有成员都是resolved，改方法的实例才是resolved
Promise.race 与all相对，可以理解为some
Promise.resolve 返回状态为resolved的实例
Promise.reject 返回状态为rejected的实例
